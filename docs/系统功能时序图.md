# 系统功能时序图

本文档基于当前项目代码整理，使用 Mermaid 时序图描述主要业务流程。

## 1. 用户注册与登录

```mermaid
sequenceDiagram
    actor 用户
    participant 前端 as Login/Register
    participant 认证接口 as AuthController
    participant 用户服务 as UserService/EmailVerificationService
    participant Redis
    participant 数据库 as PostgreSQL

    用户->>前端: 1. 填写邮箱并请求验证码
    前端->>认证接口: 2. POST /api/auth/send-email-code
    认证接口->>用户服务: 3. 生成验证码并发送邮件
    用户服务->>Redis: 4. 保存验证码和发送锁
    认证接口-->>前端: 5. 返回发送结果

    用户->>前端: 6. 提交注册表单
    前端->>认证接口: 7. POST /api/auth/register
    认证接口->>用户服务: 8. 校验验证码并创建普通用户
    用户服务->>数据库: 9. INSERT sys_user
    认证接口-->>前端: 10. 返回 JWT 和用户信息

    用户->>前端: 11. 输入用户名和密码
    前端->>认证接口: 12. POST /api/auth/login
    认证接口->>用户服务: 13. 查询用户并校验 BCrypt 密码
    用户服务->>数据库: 14. SELECT sys_user
    认证接口-->>前端: 15. 返回 JWT、角色和用户信息
    前端-->>用户: 16. 按角色进入 dashboard 或 admin
```

## 2. 数据采集

```mermaid
sequenceDiagram
    actor 用户
    participant 前端 as DataCollection.vue
    participant 站点接口 as WeatherStationController
    participant 任务接口 as CollectionTaskController
    participant 采集服务 as CollectionTaskService
    participant CMA as CMA 气象开放接口
    participant 数据库 as PostgreSQL

    用户->>前端: 1. 选择省份、站点、要素和时间范围
    前端->>站点接口: 2. GET /api/weather/stations
    站点接口->>数据库: 3. 查询 weather_stations
    站点接口-->>前端: 4. 返回站点列表

    用户->>前端: 5. 创建采集任务
    前端->>任务接口: 6. POST /api/collection-tasks
    任务接口->>采集服务: 7. 创建任务并组织 CMA 请求
    采集服务->>数据库: 8. 写入 collection_tasks
    采集服务->>CMA: 9. 请求气象观测数据
    CMA-->>采集服务: 10. 返回 JSON 数据
    采集服务->>数据库: 11. 批量写入 meteor_data 并更新任务状态
    任务接口-->>前端: 12. 返回任务结果
```

## 3. 统计分析

```mermaid
sequenceDiagram
    actor 用户
    participant 前端 as StatisticalAnalysis.vue
    participant 数据接口 as MeteorDataController
    participant 分析服务 as MeteorDataService
    participant 数据库 as PostgreSQL
    participant 图表 as ECharts

    用户->>前端: 1. 进入统计分析页面
    前端->>数据接口: 2. GET /api/meteor-data/summary
    数据接口->>分析服务: 3. 统计概览
    分析服务->>数据库: 4. 查询 meteor_data / collection_tasks
    数据接口-->>前端: 5. 返回概览指标

    用户->>前端: 6. 设置筛选条件
    前端->>数据接口: 7. GET /time-series 或 POST /correlation 或 GET /spatial-map
    数据接口->>分析服务: 8. 执行时序、相关性或空间分析
    分析服务->>数据库: 9. 聚合 meteor_data 和 weather_stations
    数据接口-->>前端: 10. 返回分析结果
    前端->>图表: 11. 渲染折线图、热力图、地图或表格
```

## 4. 普通用户报表导出

```mermaid
sequenceDiagram
    actor 用户
    participant 前端 as StatisticalAnalysis.vue
    participant 数据接口 as MeteorDataController
    participant 导出服务 as ReportExportService
    participant 分析服务 as MeteorDataService
    participant 数据库 as PostgreSQL
    participant HDFS as HDFS

    用户->>前端: 1. 点击导出 Word 或 Excel
    前端->>数据接口: 2. POST /api/meteor-data/export/analysis-docx 或 /records-xlsx
    数据接口->>导出服务: 3. 生成报表
    导出服务->>分析服务: 4. 查询分析结果、筛选项或明细记录
    分析服务->>数据库: 5. 读取 meteor_data / weather_stations / collection_tasks
    数据库-->>导出服务: 6. 返回报表数据
    导出服务->>导出服务: 7. Apache POI 生成 DOCX/XLSX
    导出服务->>HDFS: 8. 上传到 /weather-platform/exports/reports/user-{id}
    导出服务-->>数据接口: 9. 返回文件字节
    数据接口-->>前端: 10. 浏览器下载文件
```

## 9. 管理员 HDFS 存储管理

```mermaid
sequenceDiagram
    actor 管理员
    participant 前端 as StorageManagement.vue
    participant 安全框架 as Spring Security
    participant 存储接口 as StorageController
    participant HDFS服务 as HdfsService
    participant HDFS as HDFS

    管理员->>前端: 1. 打开 HDFS 工作区
    前端->>安全框架: 2. 携带 JWT 请求 /api/storage/**
    安全框架->>安全框架: 3. 校验 ROLE_ADMIN
    安全框架->>存储接口: 4. 放行请求

    前端->>存储接口: 5. GET /api/storage/overview
    存储接口->>HDFS服务: 6. 查询集群容量、SafeMode、目录状态
    HDFS服务->>HDFS: 7. getStatus / getContentSummary
    存储接口-->>前端: 8. 返回总览信息

    前端->>存储接口: 9. GET /api/storage/files?path=/
    存储接口->>HDFS服务: 10. listDirectory()
    HDFS服务->>HDFS: 11. 读取文件状态、目录汇总、块大小和副本信息
    存储接口-->>前端: 12. 返回文件列表

    管理员->>前端: 13. 上传、下载、新建目录、重命名或删除
    前端->>存储接口: 14. 调用 /api/storage/files 相关接口
    存储接口->>HDFS服务: 15. 执行 HDFS 文件操作
    HDFS服务->>HDFS: 16. 写入、读取或删除文件
    存储接口-->>前端: 17. 返回操作结果
```

## 10. 管理员 GIS 数据上传到 HDFS

```mermaid
sequenceDiagram
    actor 管理员
    participant 前端 as StorageManagement.vue
    participant 存储接口 as StorageController
    participant GIS归档 as GisWeatherHdfsArchiveService/GisWeatherTableHdfsExportService
    participant 数据服务 as WeatherStation/HistoryWeatherData Service
    participant 数据库 as PostgreSQL
    participant HDFS as HDFS

    管理员->>前端: 1. 点击归档 GIS 数据
    前端->>存储接口: 2. POST /api/storage/gis/archive
    存储接口->>GIS归档: 3. 生成 GIS 展示快照
    GIS归档->>数据服务: 4. 查询站点和要素数据
    数据服务->>数据库: 5. 读取 weather_stations/history_weather_data
    GIS归档->>HDFS: 6. 写入 /raw-data/gis/weather-gis-*.json
    存储接口-->>前端: 7. 返回 HDFS 路径

    管理员->>前端: 8. 点击上传 GIS 表数据
    前端->>存储接口: 9. POST /api/storage/gis/export-tables
    存储接口->>GIS归档: 10. 全量导出 GIS 依赖表
    GIS归档->>数据库: 11. 查询 weather_stations 和 history_weather_data
    GIS归档->>HDFS: 12. 写入 export-*/weather_stations.json、history_weather_data.json、manifest.json
    存储接口-->>前端: 13. 返回批次目录
```

## 11. 管理员 Redis 缓存管理

```mermaid
sequenceDiagram
    actor 管理员
    participant 前端 as StorageManagement.vue
    participant 存储接口 as StorageController
    participant Redis

    管理员->>前端: 1. 切换到 Redis 缓存 Tab
    前端->>存储接口: 2. GET /api/storage/cache/overview
    存储接口->>Redis: 3. INFO 和 dbSize
    存储接口-->>前端: 4. 返回版本、连接数、运行时长和 Key 数

    管理员->>前端: 5. 输入 Key 模式
    前端->>存储接口: 6. GET /api/storage/cache/keys?pattern=weather:*
    存储接口->>Redis: 7. keys / ttl / type
    存储接口-->>前端: 8. 返回 Key 列表

    管理员->>前端: 9. 确认清理缓存
    前端->>存储接口: 10. DELETE /api/storage/cache?pattern=weather:*
    存储接口->>Redis: 11. 删除匹配 Key
    存储接口-->>前端: 12. 返回删除数量
```
