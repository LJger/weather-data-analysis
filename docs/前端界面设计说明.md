# 前端界面设计说明

本文档说明当前前端页面结构、主要交互和视觉设计。项目基于 Vue 3、TypeScript、Element Plus、ECharts 和百度地图 GL 实现。

## 1. 设计目标

前端界面围绕“气象数据工作台”设计，目标是：

- 普通用户能快速完成采集、分析、GIS 查看、预测和报表导出。
- 管理员能高效完成用户治理、HDFS 文件管理和 Redis 缓存管理。
- 页面保持统一主题、统一信息层级和明确操作反馈。

## 2. 技术基础

| 类别 | 实现 |
|---|---|
| 框架 | Vue 3 + Composition API |
| 类型 | TypeScript |
| 构建 | Vite |
| UI | Element Plus |
| 图标 | `@element-plus/icons-vue` |
| 图表 | ECharts |
| 地图 | 百度地图 GL、MapVGL |
| 状态 | Pinia |
| HTTP | Axios 封装在 `src/api/http.ts` |

## 3. 信息架构

### 3.1 公共页面

- `/welcome`：系统欢迎页。
- `/login`：登录页。
- `/register`：注册页。

### 3.2 普通用户工作台

普通用户路由挂载在 `/dashboard` 下：

- 个人信息
- 数据采集
- 统计分析
- GIS 分析
- 数据预测
- 个性化服务

### 3.3 管理员工作台

管理员路由挂载在 `/admin` 下：

- 用户管理
- 存储管理

## 4. 权限和跳转

前端路由守卫读取 `localStorage`：

- `token`：判断是否登录。
- `role`：判断普通用户或管理员。

跳转规则：

- 未登录访问业务页，跳转 `/login`。
- 普通用户访问 `/admin/**`，跳转 `/dashboard`。
- 管理员访问 `/dashboard/**`，跳转 `/admin`。
- 已登录用户访问登录/注册页，按角色回到对应首页。

## 5. 页面设计

### 5.1 登录与注册

登录页和注册页采用业务型认证页面：

- 登录页提供用户名、密码、记住状态和角色分流。
- 注册页提供邮箱验证码、密码校验、手机号、地区、机构等资料输入。
- 注册接口只创建普通用户，管理员账号由后台管理。
- 认证错误通过统一工具 `src/utils/authError.ts` 归一化展示，避免后端原始错误直接暴露给用户。

### 5.2 用户布局

`Layout.vue` 提供普通用户主框架：

- 左侧导航。
- 顶部标题和用户入口。
- 中间业务内容区。
- 系统设置入口与主题切换。

### 5.3 数据采集页

`DataCollection.vue` 面向采集任务管理：

- 任务统计。
- 站点、省份、要素、时间范围选择。
- 创建任务。
- 任务列表、详情、状态更新和删除。
- CMA API 调试/模拟入口。

### 5.4 统计分析页

`StatisticalAnalysis.vue` 面向数据分析：

- 顶部概览卡片。
- 时间序列分析。
- 相关性热力图。
- 空间分析。
- 明细数据表格。
- Word 分析报告导出。
- Excel 明细导出。

导出行为：

- 前端仍触发浏览器下载。
- 后端同步归档到 HDFS `/exports/reports/user-{id}`。

相关性热力图已调整底部空间，避免 x 轴标签和视觉组件重叠。

### 5.5 GIS 分析页

`WeatherMapView.vue` 以地图为核心：

- 站点 Marker 展示。
- 温度、湿度、降水要素切换。
- 观测时间选择。
- 点选地图查询最近站点。
- 站点详情、气象数据、经纬度和距离信息弹窗。

点选查询弹窗使用分区卡片展示：

- 最近站点。
- 气象数据。
- 位置坐标。

### 5.6 数据预测页

`DataPrediction.vue` 提供预测配置和结果展示：

- 选择要素、站点、时间范围、算法。
- 线性回归、移动平均、指数平滑。
- 预测曲线和历史曲线。
- MAE、RMSE、R² 指标。
- 预警阈值提示。

### 5.7 个性化服务页

`PersonalizedService.vue` 集中管理：

- 用户数据概览。
- 最新数据表格。
- 要素分布图。
- 主题配置。
- 默认分析偏好。
- 快捷入口。

页面布局已经调整，使右侧卡片与左侧区域对齐。

### 5.8 管理员用户管理

`UserManagement.vue` 面向用户治理：

- 用户统计。
- 用户列表。
- 新增、编辑、删除。
- 角色切换。
- 重置密码。
- 批量操作。

### 5.9 管理员存储管理

`StorageManagement.vue` 是当前重点页面，分为两个 Tab。

#### HDFS 工作区

HDFS 页面不再使用模拟数据，直接调用真实接口：

- `/api/storage/overview`
- `/api/storage/files`
- `/api/storage/files/upload`
- `/api/storage/files/download`
- `/api/storage/files/mkdir`
- `/api/storage/files/rename`
- `DELETE /api/storage/files`
- `/api/storage/init`
- `/api/storage/gis/archive`
- `/api/storage/gis/export-tables`

界面能力：

- HDFS 状态标签。
- 总容量、已用、可用、Redis 内存概览。
- SafeMode 和平台目录未初始化提示。
- 平台目录初始化按钮。
- 面包屑路径。
- 文件/目录表格。
- 展开行展示 HDFS 元数据。
- 上传、下载、重命名、删除。

HDFS 元数据展示：

- 文件大小。
- HDFS 占用。
- 副本数。
- 块大小。
- 估算块数。
- Owner、Group、权限、修改时间。
- 目录递归文件数和子目录数。

#### Redis 缓存

Redis 页面展示：

- Redis 版本。
- 连接数。
- 运行时长。
- 匹配 Key 数。
- Key 类型和 TTL。
- 按模式查询和清理。

## 6. 主题系统

主题系统由 `src/stores/theme.ts` 和 `src/styles/theme.css` 管理。

支持：

- 预设主题。
- 浅色、深色、跟随系统。
- 主色、字体大小、圆角。
- 动画开关。
- localStorage 持久化。

业务页面尽量使用 CSS 变量：

- `--theme-primary`
- `--theme-background`
- `--theme-surface`
- `--theme-text`
- `--theme-text-secondary`
- `--theme-border`

## 7. 组件与交互规范

- 常用操作使用 Element Plus 图标按钮。
- 删除、清理、账号注销等风险操作必须二次确认。
- 表格操作保持固定右侧操作区。
- HDFS、Redis、报表导出等异步操作必须显示 loading。
- 空状态和错误状态应显示明确原因。
- 页面文字保持业务语义，不使用“模拟数据”描述真实接口页面。

## 8. 构建和验证

```bash
cd front_end
npm install
npm run type-check
npm run build
```

当前构建可能出现 Vite chunk 体积提示，这是打包体积提示，不代表构建失败。
